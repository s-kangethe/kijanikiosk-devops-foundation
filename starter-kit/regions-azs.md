The selected region is af-south-1(Cape Town). Capetown is the closest AWS region to East Africa as most users are in Kenya and neighboring countries. This geographic closeness promotes low latency thus enabling faster response times for users in Kenya. AWS in Africa has 4 Edge Network locations, and one of them is in Nairobi, Kenya which will help to reduce delays by helping in caching and routing traffic efficiently.The AWS Africa region has 3 Availability Zones(AZs): af-south-1a,af-south-1b, af-south-1c which are physically separate,,,independent networks and are isolated failure domain.I would have the public layer/subnets with Multi-AZ, API-kk-api, deploy EC2 instances in AZ-a, AZ-b and place behind a load balancer to ensure traffic is distributed and there is no single point of failure.
The private layer/subnets with Multi-AZ handle backend and database,Payments-kk-payments, deploy backend services in AZ-a, AZ-b, to ensure backend continues even if one AZ fails and database later with primary database on AZ-a and a standby database on AZ-b to allow automatic failover to ensure high availability. With the use of Multi-AZ, if one AZ fails, it shits to a healthy AZ which ensures the system remains available.
